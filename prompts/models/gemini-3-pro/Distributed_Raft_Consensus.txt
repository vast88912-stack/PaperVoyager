**Project:** Raft Consensus Simulator (Gold Standard)
**Role:** Senior Creative Technologist.
**Objective:** Build a high-complexity, single-file React+TS teaching dashboard that simulates the Raft Distributed Consensus Protocol (Leader Election, Log Replication).

**Global UX Spec:**
- Layout: Landing Page (Hero) -> Dashboard (Sidebar + Main + Inspector).
- Theme: Scientific Dark Mode (Server Rack/Terminal aesthetic).
- Navigation: Scenario Selector (Normal, Partition, Failures).
- Inspector: "Global Log" vs "Node Logs".

**Module Plan (Landing + 5 Modules):**
1. **Module 0: Hero / The Split Brain**
   - Visual: 5 servers agreeing on a value, then a network partition happens, and chaos (disagreement) occurs.
   - Text: "In a distributed world, Truth is relative. Consensus makes it absolute."

2. **Module 1: Node States & Timers**
   - Concept: Follower, Candidate, Leader.
   - Visualization: 5 Nodes (Circles). Each has a "Election Timeout" progress bar.
   - Interactive: Watch them countdown. First to timeout becomes Candidate.

3. **Module 2: Leader Election**
   - Concept: Voting.
   - Animation: Candidate sends "RequestVote" RPCs (dots flying). Nodes reply.
   - Scenario: Kill the Leader. Watch a new election trigger.

4. **Module 3: Log Replication**
   - Concept: Appending entries.
   - Interactive: Client sends "SET X=5".
   - Animation: Leader gets it -> Sends "AppendEntries" to Followers -> Followers Ack -> Leader Commits -> Leader notifies Followers to Commit.
   - Visual: The "Log" inside each node updates step-by-step.

5. **Module 4: Network Partition (The Brain Split)**
   - Concept: Majority Quorum.
   - Controls: "Cut Network" tool. Sever the link between 2 nodes and the other 3.
   - Simulation: Old Leader is in the minority partition. It accepts writes but CANNOT commit. New Leader rises in the majority.
   - Recovery: Heal the network. Watch the Old Leader step down and overwrite its uncommitted log.

6. **Module 5: The Consensus Challenge**
   - Task: "Force a localized rollback."
   - Goal: Manipulate the network (disconnect/reconnect) to make a node accept a log entry that eventually gets overwritten.
   - Scoring: Understanding Raft safety properties.

**Scoring & Gamification:**
- **SysAdmin Rank**: Earned by maintaining uptime.
- **Chaos Engineer**: Earned by creating complex failure scenarios.

**Implementation Notes:**
- **Simulation Loop**: Use a `useEffect` loop or `requestAnimationFrame` to drive the state machine ticks.
- **Visuals**: Flying message packets are crucial. Use `framer-motion` layout animations for packets moving between nodes.
- **State**: Strictly model the Raft state (term, votedFor, log, commitIndex).
